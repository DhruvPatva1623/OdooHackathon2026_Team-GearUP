# Contributing to OdooHackathon2026_Team-GearUP

Thank you for contributing! This project follows an open, collaborative process. Please read the guidelines below before submitting code or documentation.

## How to Contribute

1. Fork the repository and create a feature branch:
   ```bash
   git checkout -b feature/<your-feature-name>
   ```
2. Keep commits focused and descriptive.
3. Push your branch to your fork and open a pull request.

## Code Style

- Backend: follow standard Django and Python conventions.
- Frontend: keep React components readable and use consistent JSX formatting.
- Use existing patterns where possible.

## Commit Messages

Use clear, concise commit messages. Prefer a format like:

- `feat: add user login flow`
- `fix: correct API pagination bug`
- `docs: update README installation instructions`

## Testing

- Backend: ensure Django migrations run cleanly and API endpoints respond correctly.
- Frontend: ensure the app starts and renders as expected with `npm run dev`.

## Pull Requests

Your PR should:

- describe the problem and solution
- reference relevant issues, if any
- include screenshots or examples for UI changes
- be reviewable and limited in scope

## Issues

If you find a bug or want to request a feature, open an issue with:

- expected behavior
- actual behavior
- steps to reproduce
- environment details (browser, OS, etc.)
