# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly by opening a private issue or contacting the maintainers directly.

Do not disclose the vulnerability publicly until it has been addressed.

## Supported Versions

This project currently supports the latest version of the repository in the `main` branch.

## Security Best Practices

- Keep dependencies up to date.
- Do not commit secrets or API keys to the repository.
- Use secure authentication and encrypted communication for production deployment.

## Response Policy

The maintainers will acknowledge security reports within a reasonable time and work to remediate confirmed vulnerabilities promptly.

## Contact

Open an issue titled `Security: <brief description>` or use the repository maintainer contact information if available.
